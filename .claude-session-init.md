## 🎯 8 Giải Pháp Tiết Kiệm Token + Data Integrity (BẮTBUỘC)

| # | Giải Pháp | Tiết Kiệm | Cách Làm |
|---|-----------|-----------|---------|
| 1️⃣ | Không include chat cũ | 10-15% | Mỗi request mới → không lặp request trước |
| 2️⃣ | Batch request liên quan | 10-20% | Group 2-3 task giống nhau → 1 request |
| 3️⃣ | Grep trước, đọc sau | 20-30% | Luôn grep index.md trước khi quyết định đọc file |
| 4️⃣ | Ưu tiên file nhỏ | 15-25% | style → trang-thai → index → arc → nhan-vat → chuong |
| 5️⃣ | Chỉ dùng chuong/ cuối | 30-40% | Chỉ đọc chương gốc khi thực sự cần |
| 6️⃣ | Ghi file, show status | 10-20% | Response dài → ghi file, chat show link + summary |
| 7️⃣ | Combine dependent tasks | 5-15% | Task phụ thuộc → 1 request sequential |
| 8️⃣ | **Data Integrity Priority** | **0%** | **chuong/ = nguồn chân lý, verify mâu thuẫn** |

**Tổng Tiết Kiệm:** ~30-45% tokens/session + **100% data accuracy**
